# Cuándo Usar Fee-Bump Transactions

## Casos Ideales

### Onboarding de Usuarios Nuevos
Cuando tu dApp onboardea usuarios que no tienen XLM y no quieres que tengan que comprarlo antes de usar tu aplicación.

### Aplicaciones DeFi
- Swaps de tokens
- Lending protocols
- Staking/unstaking

### NFT Marketplaces
- Minting de NFTs
- Ventas/compras de NFTs
- Listados/remociones

### Games y Aplicaciones Social
- Transacciones frecuentes de bajo valor
- micro-transacciones
- Acciones in-game que requieren fees

## Cuándo NO Usar

### Transacciones de Alto Valor
Si la transacción involucra grandes cantidades de activos, puede ser mejor que el usuario pague sus propias fees por seguridad y confianza.

### Casos de Uso Regulatorios
En jurisdicciones donde el modelo de "sponsor" pueda tener implicaciones regulatorias.

### Cuando el Usuario Ya Tiene XLM
Si tu base de usuarios ya tiene XLM, el patrón tradicional puede ser más simple.

## Consideraciones

- El relayer debe ser confiable o descentralizado
- El sponsor debe tener fondos suficientes
- Latencia adicional vs. transacciones directas
- Complejidad adicional en la UX
